When would you want to create a remote repository rather than keeping all your work local?

I think when I want to share my project with someone else or when I want to work on my project on different computers.

为何你希望始终手动地拉取更改，而不是让 Git 自动与你的远程版本库保持同步？

始终保持同步有时可能会遇到一些网络延迟上的问题，不对啊…能始终同步不是挺好的嘛……

说明Fork、克隆和分支之间的不同。何时使用某种功能而不是另一种？

Fork 是 GitHub 提供的一种功能，可以将他人的代码库给自己使用

拥有远程分支的上次已知状态的副本（存储在本地）有何好处？

可以判断各种合并的情况

如何在不使用 Git 或 GitHub 的情况下开展协同？哪种做法更容易？哪种做法更困难？

使用电子邮件发送文件，使用 U 盘到处复制。使用 Git 或 GitHub 更加容易。

何时你会想在独立的分支中而不是直接在 master 中进行更改？两种方法各有何好处？

当我需要进行一些不确定的，或者实验性的更改的时候，不能影响 master 中的内容，会需要一个独立的分支来更改或者实验。
